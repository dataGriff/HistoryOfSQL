Select Statement

Below is the template code for a basic select statement in SQL.

USE DatabaseName;
GO

SELECT
[ColumnA], [ColumnB], [ColumnC]
FROM [SchemaName].[TableName];

The template shows

  • The columns you want to retrieve go into your SELECT clause.
  • The object(s) that contain your columns go into your FROM clause.

Which is equivalent to...

USE DatabaseName;

The example shows us that using an asterisk in place of the columns will return all columns from the object(s) in the from clause. However tempting this may be do not use this practice in production! Specify the columns you lazy dogs! If the column order changes it can really mess things up if you continue using the asterisk. Also... don't be greedy! Taking only the columns you need is far more efficient.

Now we're going to restrict our output using the WHERE clause

This example restricts the result set by returning only those rows where value of the DogColour column is brown.

USE dktest;
GO

SELECT
[DogName], [DogBreed], [DogAgeYears]
FROM [import].[PoundDogs]
WHERE [DogColour] = 'Brown';

This example restricts the result set by returning only those rows where value of the DogColour contains the characters "brown".

USE dktest;
GO

SELECT
[DogName], [DogBreed], [DogColour]
FROM [import].[PoundDogs]
WHERE [DogColour] LIKE '%Brown%';

This example restricts the result set by returning only those rows where value of the DogColour is white and the value of DogBreed is Staffordshire Bull Terrier.

USE dktest;
GO

SELECT
[DogName], [DogAgeYears], [DateArrived]
FROM [import].[PoundDogs]
WHERE [DogColour] = 'White'
AND [DogBreed] = 'Staffordshire Bull Terrier';

This example restricts the result set by returning only those rows where value of the DogColour is Brown or the value of DogBreed is Staffordshire Bull Terrier.

USE dktest;
GO

SELECT
[DogName], [DogAgeYears], [DateArrived], [DogColour], [DogBreed]
FROM [import].[PoundDogs]
WHERE [DogColour] = 'Brown'
OR [DogBreed] = 'Staffordshire Bull Terrier';

This example restricts the result set by returning only those rows where the dog name is Mindy, Terry or Vinny.

USE dktest;
GO

SELECT
[DogName], [DogBreed], [DogColour], [DogAgeYears]
FROM [import].[PoundDogs]
WHERE [DogName] IN ( 'Mindy', 'Terry', 'Vinny') ;

What have we learned from our examples?

  • You can restrict the rows returned using the WHERE clause.
  • The WHERE clause can contain logical conjugates AND and OR to combine predicates.
  • The WHERE clause can contain different operators for implementing criteria, =, LIKE and IN are the positive matching operators and <>, NOT LIKE, and NOT in are the negative matching operators.
  • You can restrict the columns returned by limiting the number of columns in the SELECT clause.
